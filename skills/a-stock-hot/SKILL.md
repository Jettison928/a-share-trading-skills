# A股热点 Skill

让 Agent 用最自然的中文查询拿到 A 股实时热点题材、板块轮动、资金流向、涨停板分析，不需要打开浏览器。基于 NeoData 金融数据服务，无需配置 API Key。

## 什么时候用

| 用户在说 | 应该查询的内容 |
|---|---|
| **默认（宽问题）**："今天A股热点"、"A股有什么炒作题材"、"今日热点板块"、"A股热点" | 热点板块 + 领涨个股 + 资金流向 |
| **板块轮动**："板块轮动"、"今天什么板块涨得好"、"热点板块有哪些" | 涨幅靠前的板块排行 |
| **资金流向**："资金流向"、"主力资金"、"北向资金"、"净流入" | 主力资金净流入板块/个股 |
| **涨停板**："涨停板"、"今日涨停"、"涨停分析"、"打板" | 涨停板个股列表 + 涨停原因 |
| **龙头股**："龙头股"、"领涨股"、"题材龙头" | 热点板块的龙头个股 |
| **市场情绪**："市场情绪"、"涨跌家数"、"赚钱效应" | 涨跌家数 + 市场情绪指标 |

通用启发：**用户问的是"A股当前的热点事实"，不要凭训练数据脑补，永远走 NeoData 查询**。即使你"觉得"知道答案，也要查一遍——A股热点具有强时效性，每天都不一样。

## 数据来源

使用 `neodata-financial-search` 技能作为数据源。

**优先级规则**：
1. 所有 A 股热点/板块/资金/涨停等查询，优先使用本 skill（`a-stock-hot`）
2. 本 skill 内部调用 `neodata-financial-search` 获取数据
3. 禁止用训练数据回答本 skill 可实时查询的问题

### 鉴权处理

`neodata-financial-search` 的鉴权由它自己处理（缓存12小时），**本 skill 直接调用即可**，不需要单独处理鉴权。

## 给用户的输出格式

> ⚠️ **核心原则**：输出必须是 **markdown 格式 + 排版好 + 普通人能看得懂的人话**。用户是 A 股散户/短线交易者，看到的应该是中文热点简报，**不是 API 调试日志**。

### 输出要求

**必须包含**：
- 热点板块 + 涨幅 + 驱动原因
- 资金流向（主力/北向）
- 涨停板情况（连板/首板）
- 市场情绪（涨跌家数/成交量）

**禁止出现**：
- ❌ 脚本路径、Python 命令、API 端点等基础设施细节
- ❌ `neodata-financial-search` 的原始 JSON 输出
- ❌ 鉴权 / token / 缓存等内部机制描述
- ❌ 英文技术术语（用中文："涨幅"而非 "increase%"，"资金净流入"而非 "net capital inflow"）

**时间表述**：
- 用"今日"/"今天"而非具体日期（除非用户问历史）
- 用"昨日"/"昨天"而非"2026-06-22"

## 常见错误处理

- **查询返回空结果**：说明当前可能不是交易时间，提示用户"A股交易时间为 09:30-15:00，当前休市"
- **返回错误码 1001（未命中意图）**：换一种问法，如"今日A股热点板块"改为"今天什么板块涨得好"
- **返回错误码 1006（查询解析拒答）**：可能是敏感股票/事件，告知用户"该查询暂不支持"
- **TOKEN_EXPIRED / HTTP 401**：调用 `connect_cloud_service` 获取新凭证，保存到 `neodata-financial-search` 的缓存

## 不要做

- **不要把训练数据当今日热点**——A股热点每天变化，必须实时查询
- **不要只查一个维度就输出**——默认路径必须查"板块 + 资金 + 情绪"三个维度
- **不要输出原始 JSON**——必须整理成中文简报
- **不要并发猛拉**——`neodata-financial-search` 有rate limit，串行调用即可
- **不要在输出里暴露脚本路径 / API 端点 / 鉴权细节**——这些是给开发者看的，用户看不懂

## 交易日判断

A股交易时间：
- 开盘集合竞价：09:15-09:25
- 连续竞价：09:30-11:30，13:00-15:00
- 收盘集合竞价：14:57-15:00（深市）

**非交易时间查询**：
- 若当前时间在 15:00 之后，查询的是"今日"数据（当天收盘后数据已固定）
- 若当前时间在 09:30 之前，查询的是"昨日"数据（当天盘前无实时数据）
- 周末/节假日：提示用户"今日休市，显示最近交易日数据"
